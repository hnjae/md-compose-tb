# Documentation

This directory contains project documentation for MD Compose for Thunderbird.

Use these subdirectories by document purpose:

- `docs/research/`: investigation notes, platform behavior checks, API feasibility notes, and PoC result records used before product behavior is finalized.
- `docs/spec/`: user-facing product behavior and externally visible decisions. Update these documents before changing behavior.
- `docs/architecture/`: internal structure, package boundaries, dependency choices, and major implementation strategies.

Keep documents concise and technical. Do not hard-wrap ordinary prose. Use Mermaid for diagrams when a diagram is useful.

Research documents may describe uncertainty and open questions. Spec documents should describe committed product behavior. Architecture documents should explain implementation structure and tradeoffs without duplicating user-facing behavior details from specs.
